# Repositório de Documentação do Projeto

Este repositório tem como objetivo centralizar e versionar todos os documentos produzidos ao longo do desenvolvimento do projeto, desde a Sprint 1 até a Sprint 4.

## 📁 Estrutura

A pasta `Documentos` contém os principais arquivos gerados até o momento, incluindo:

- Planejamento de testes  
- Mapas mentais  
- Collections do Postman  
- Artefatos de backlog e outras documentações relevantes

O uso de branches está sendo adotado para facilitar o controle de versão e a organização das entregas ao longo do projeto.

## 🚀 Objetivo

Manter um histórico organizado e versionado dos materiais produzidos, facilitando a colaboração, o acompanhamento da evolução e a continuidade do desenvolvimento nas próximas etapas.

## 👤 Modalidade

Este repositório é individual e faz parte das atividades avaliativas da disciplina.

## 🛠 Tecnologias e Ferramentas

- Git e GitHub para versionamento
- Postman para testes de API
- Ferramentas de mapeamento mental
- Documentação técnica em formato `.pdf`, `.docx` e outros
